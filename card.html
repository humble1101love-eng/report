高德-千问卡片数据流转总结

Amap & Qwen Card Data Flow Architecture
1
请求入口 (Request Entry) Gateway
接收来自千问大模型的 MCP (Model Context Protocol) 请求，作为流量网关进行初步校验与分发。
McpApiController Route Dispatch
2
业务逻辑处理 (Business Logic) Core Service
执行核心业务规则，查询订单状态或POI信息，组装基础业务对象。
BizService FlowHandler Returns: FlowHandlerBO
3
卡片构建 (Card Rendering) Template Engine
将通用的业务 BO 转换为前端可渲染的卡片数据结构，适配千问卡片协议规范。
CardRender OrderCardService Output: CardDataWrapper
4
持久化存储 (Persistence) Database
将生成的卡片快照存入数据库，建立 cardId 与业务订单的映射关系，支持后续追溯与更新。
QwenCardService QwenCardDomainService Action: Save & Generate ID
5
消息推送 (Push Notification) RPC / HSF
通过异步消费者或同步接口，将构建好的卡片数据推送到千问服务端。
ChatRecordServiceConsumer Protocol: HSF Interface
6
终端渲染 (Client Rendering) UI Layer
千问客户端接收 JSON 数据，根据模板 ID 解析并渲染出可视化的交互式卡片。
Qwen Client JSON to UI
7
动态更新 (Dynamic Update) Event Driven
监听订单状态变更事件（如：已接单、送达），触发卡片局部刷新或全量替换，保持信息实时性。
Status Change Trigger Re-render & Push
